# vibe-app

Куратор Яндекс.Музыки: автодетект AI-треков, чистка библиотеки, локальный архив.
Android-приложение (sideload APK). Планировщик находит новые треки в библиотеке ЯМ →
детектит AI → чистит библиотеку по настроенному режиму → скачивает чистое → пополняет архив.

> Публичная часть проекта (для CI/сборки). Спецификации, приватная инфраструктура и
> секреты живут отдельно и в этот репозиторий не попадают.

## Статус

Ранняя разработка, чистые JVM-модули (Android-слой поверх — позже):
- **yandex-слой** — подпись запросов, дешифровка аудио (AES-128-CTR), метаданные, get-file-info,
  бэкап/restore (dry-run). Крипто- и подписной узлы проверены байт-в-байт против референс-реализации.
- **детектор** — каскад с hard gate (§10): база slopless → эвристики метаданных → [аудио-плагин].
  Аудио-слой отложен (см. атрибуции). Линейный инференс сверен с Python-эталоном (golden).
- **state machine** трека (§5) + схема SQLite-индекса.

## Сборка и тесты

```bash
./gradlew build     # сборка + юнит-тесты
./gradlew test      # только тесты
```

Требуется JDK 17+. Gradle подтягивается через wrapper — ставить отдельно не нужно.

## Структура

```
src/main/kotlin/dev/humanonly/yandex/
  Signing.kt       — HMAC-подпись get-file-info (DEFAULT_SIGN_KEY, публичная константа клиента ЯМ)
  TrackCrypto.kt   — дешифровка AES-128-CTR (IV=0), seekable-срезы для докачки
src/test/kotlin/dev/humanonly/yandex/
  SigningTest.kt      — подпись против golden-фикстуры (офлайн, детерминированный)
  TrackCryptoTest.kt  — round-trip AES-CTR + дешифровка срезов с блочного смещения
```

## Технологии

Kotlin/JVM · kotlinx-serialization · JUnit 5. Android-слой (WorkManager, Media3, SQLite)
добавляется поверх этого модуля позже.

## Сторонние компоненты и атрибуции

- **База AI-артистов — [`alexeyfv/slopless`](https://github.com/alexeyfv/slopless) (GPL-3.0).**
  Совместима с лицензией проекта. Сам список артистов в этот репозиторий **не включён** —
  загружается в рантайме отдельной поставкой.
- **Аудио-детектор — [`deezer/ismir25-ai-music-detector`](https://github.com/deezer/ismir25-ai-music-detector)
  (CC BY-NC 4.0 + патентные заявки на метод, дек. 2024).** Метод FFT-«fakeprint» в публичной
  сборке **НЕ реализован и не распространяется** (несовместим с GPL/свободным распространением и
  патентно обременён). Детекция здесь — hard gate (slopless) + эвристики метаданных; аудио-слой —
  на усмотрение пользователя вне этого репозитория. Научная работа:
  D. Afchar et al., «A Fourier Explanation of AI-music Artifacts», ISMIR 2025.

## Дисклеймер

Инструмент использует неофициальный доступ к API Яндекс.Музыки и нарушает ToS сервиса —
использование на свой риск. Не аффилирован с Яндексом, Deezer или slopless.
