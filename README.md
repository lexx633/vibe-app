# humanonly-app

Куратор Яндекс.Музыки: автодетект AI-треков, чистка библиотеки, локальный архив.
Android-приложение (sideload APK). Планировщик находит новые треки в библиотеке ЯМ →
детектит AI → чистит библиотеку по настроенному режиму → скачивает чистое → пополняет архив.

> Публичная часть проекта (для CI/сборки). Спецификации, приватная инфраструктура и
> секреты живут отдельно и в этот репозиторий не попадают.

## Статус

Ранняя разработка. Сейчас — **yandex-слой** как чистый JVM-модуль:
подпись запросов, дешифровка аудио (AES-128-CTR), метаданные, get-file-info.

Криптографический и подписной узлы проверены байт-в-байт против референс-реализации
(golden-сверка) — на этом основано решение писать yandex-слой на Kotlin-нативе.

## Сборка и тесты

```bash
./gradlew build     # сборка + юнит-тесты
./gradlew test      # только тесты
```

Требуется JDK 17+. Gradle подтягивается через wrapper — ставить отдельно не нужно.

## Структура

```
src/main/kotlin/dev/humanonly/yandex/
  Signing.kt       — HMAC-подпись get-file-info (DEFAULT_SIGN_KEY, публичная константа клиента ЯМ)
  TrackCrypto.kt   — дешифровка AES-128-CTR (IV=0), seekable-срезы для докачки
src/test/kotlin/dev/humanonly/yandex/
  SigningTest.kt      — подпись против golden-фикстуры (офлайн, детерминированный)
  TrackCryptoTest.kt  — round-trip AES-CTR + дешифровка срезов с блочного смещения
```

## Технологии

Kotlin/JVM · kotlinx-serialization · JUnit 5. Android-слой (WorkManager, Media3, SQLite)
добавляется поверх этого модуля позже.
